01 · Diagnóstico
El problema no es falta de data. Es falta de integración.
Hernán ya tiene las fuentes correctas. El problema es que cada una vive en su propio silo: HouseCanary en una tab, Zillow en otra, los costos en una planilla, la oferta futura en ningún lado. Cada análisis tarda horas que deberían tomar segundos.
🏷️
HouseCanary
Solo valuación
Excelente para ARV y renta estimada, pero no integra costos de obra ni pipeline. Los datos no salen de la plataforma.
🏘️
Zillow / Redfin
Comps manuales
Hay que cruzar las ventas a mano. Sin automatización ni radio dinámico. No hay cálculo de spread integrado.
📊
Zonda (evaluando)
Caro y oversized
Orientado a builders grandes. No tiene API abierta práctica para el uso operativo de Dixa. ROI difícil de justificar.
🔨
Costos de obra
Estimados "a ojo"
No existe herramienta que parametrice el costo por sqft con benchmarks actualizados para el mercado de Florida.
🏗️
Oferta futura
No existe
Sin visibilidad de nuevos permisos de construcción en el ZIP analizado. Decisiones tomadas sin conocer la competencia real.
📋
Vista integrada
No existe
Todo vive en tabs de browser y planillas de Excel. No hay un dashboard único que consolide el análisis de un deal.
🔍 El dolor real: El equipo de DIXA invierte horas en análisis que, con la integración correcta, deberían tomar menos de 60 segundos. Cada deal que tarda en analizarse es una ventana de oportunidad que se cierra.
02 · La Solución
DIXA Deal Analyzer
Una plataforma web liviana que centraliza toda la inteligencia de un deal en un solo dashboard. Ingresás una dirección y en segundos obtenés ARV, comps, costo estimado de obra, pipeline de competencia y una recomendación Go/No-Go automática.
⚡

De una dirección a un análisis completo en <60 segundos

El sistema consolida automáticamente HouseCanary, Zillow, datos de permisos municipales y benchmarks de costos de construcción de Florida en un solo panel.

🏷️
ARV Automático
Valuación vía HouseCanary API con AVM, valor por condición e historial de ventas.
📊
Comps Dinámicos
Ventas recientes y rentas del radio con $/sqft y días en mercado, cruzadas de Zillow + Redfin.
💰
Costos de Obra
Módulo parametrizado con RSMeans + histórico Dixa: tipo de construcción, sqft y zona de Florida.
🚦
Go / No-Go
Cálculo instantáneo de margen bruto = ARV − (terreno + obra + soft costs + carry). Semáforo automático.
🖥️
Demo disponible

Ya existe un prototipo funcional

El mockup del Deal Analyzer ya está construido y muestra exactamente cómo se vería la experiencia: ingreso de dirección, análisis de ARV, tabla de comps, estructura de costos, KPIs clave y recomendación final.

03 · Roadmap de Desarrollo
Tres fases, valor incremental desde el día uno
El producto se construye en capas: primero el núcleo de análisis rápido (MVP), luego inteligencia de mercado, y finalmente automatización con IA. Cada fase entrega valor inmediato antes de pasar a la siguiente.
F1

MVP — Deal Screening Dashboard

⏱ Semanas 1–2 Prioritario
Un panel donde Hernán ingresa una dirección y obtiene automáticamente todo lo que necesita para decidir si vale la pena profundizar en el deal. Sin pestañas, sin planillas.
📊 ARV & Renta Estimada Integración directa con HouseCanary API (/v2/property/value, /v2/property/rental_value). AVM + valor por condición + historial.
🏘️ Comps Automáticos Ventas recientes y rentas en radio configurable. Zillow API + Redfin data. Tabla con precio, $/sqft y días en mercado.
🔨 Estimador de Obra Módulo parametrizado con RSMeans/BuildZoom o modelo propio Dixa. Input: tipo de construcción, sqft, zona FL.
🚦 Go/No-Go Automático Margen bruto = ARV − (terreno + obra + soft costs + carry). Semáforo verde/amarillo/rojo configurable por Hernán.
F2

Pipeline & Inteligencia de Mercado

⏱ Semana 3 Diferenciador clave
Visibilidad total de la oferta futura: permisos aprobados, proyectos en construcción y competencia directa high-end en cada ZIP analizado. Mapas interactivos y scoring de submercado.
🏗️ Oferta Futura Integración con portales públicos de permisos de Broward, Miami-Dade y Palm Beach. ETL automatizado.
🗺️ Mapa Interactivo Heatmap de actividad de permisos, comps y densidad de inventario por ZIP. Vista geográfica del mercado.
📈 Scoring de Submercado Algoritmo que rankea zonas por spread (ARV vs. costo), velocidad de absorción y pipeline de competencia.
📉 Análisis de Absorción Velocidad histórica de venta por ZIP y segmento de precio. Proyección de tiempo en mercado para el deal.
F3

Inteligencia Artificial & Automatización

⏱ Semana 4 Roadmap futuro
El sistema aprende y trabaja solo: agente conversacional de deals, reportes automáticos listos para inversores, y alertas en tiempo real cuando aparece actividad relevante en los ZIPs que Dixa está monitoreando.
🤖 Agente "Preguntale al Deal" Subí una dirección o un deal memo y el sistema responde: spread estimado, comps, pipeline, riesgo. Lenguaje natural.
📑 Reportes Automáticos PDF/Word de viabilidad listo para presentar a inversores o socios, con toda la data consolidada en minutos.
🔔 Monitoreo Continuo Alertas automáticas cuando se aprueba un nuevo permiso en un ZIP watchlisted. Dixa se entera antes que la competencia.
🧠 Modelo Predictivo Predicción de ARV y absorción entrenado con histórico de ventas de Florida. Mejora con cada deal procesado.
04 · Factibilidad Técnica
Integraciones validadas y viables
Cada integración del stack fue evaluada en términos de disponibilidad de API, costo operativo y dificultad de implementación. El 80% son green-light desde el día uno.
HouseCanary
✅ API lista
REST API bien documentada y SOC 2 compliant. Endpoints /v2/property/value y /v2/property/rental_value. Base del módulo de ARV.
💲 ~$0.50–2/lookup 🟢 Dificultad baja
Zillow API
✅ Disponible
APIs públicas + datasets para Zestimates y ventas recientes. Free tier con opción a partner plan para mayor volumen y calidad de datos.
💲 Free / partner 🟢 Dificultad baja
Redfin Data
⚠️ Sin API oficial
No tiene API pública, pero tiene data feeds descargables y existen integraciones alternativas via scraping controlado o servicios intermediarios.
💲 Variable 🟡 Dificultad media
Permisos FL — Estado por Condado
⚠️ Mixto — investigado
Realizamos una investigación técnica profunda de las APIs públicas de los tres condados target. El resultado no es uniforme — y es importante saberlo desde el arranque:
✅ Miami-Dade
API confirmada y funcional. ArcGIS FeatureServer en AGOL (services.arcgis.com). 2,000+ permisos reales por ZIP, últimos 2 años. Datos en vivo.
⚠️ Broward
Sin API REST pública estable. bcgis.broward.org (503 frecuente), ningún FeatureServer accesible encontrado tras investigación exhaustiva. Fase 1: datos mock realistas. Roadmap: integración vía ATTOM Data o acuerdo directo con el condado.
⚠️ Palm Beach
Solo datos agregados disponibles públicamente (estadísticas trimestrales por zona, no permisos individuales). Alternativas evaluadas: ATTOM Data API (~$200–400/mes) para cobertura completa de los 3 condados.
💡 Solución de largo plazo: ATTOM Data API cubre los 3 condados con permisos individuales a ~$200–400/mes. Puede incorporarse como upgrade en Fase 2 o Fase 3 sin cambiar la arquitectura del sistema.
💲 Miami-Dade: Gratuito · Broward/PB: Mock F1 → ATTOM $200–400/mes 🟡 Complejidad mixta por condado
Costos de Obra
⚠️ Modelo propio
RSMeans/Gordian tienen API pero son caros. La alternativa más viable y diferenciadora: modelo propio de Dixa alimentado con histórico de proyectos reales.
💲 Modelo propio = $0 🟡 Dificultad media
Zonda
⚠️ Premium / Fase 3
API limitada y orientada a enterprise. Costo premium alto. Evaluada para Fase 3 si los portales públicos no alcanzan el nivel de detalle requerido.
💲 Premium ($$) 🔴 Dificultad alta
